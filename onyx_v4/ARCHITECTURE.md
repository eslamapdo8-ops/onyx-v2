# 🧬 Onyx V4 — HD NCO Array

## ⚡ 1. المبدأ

مصفوفة (Array) من **N** من مذبذبات NCO الرقمية تعمل بالتوازي، كل منها يستقبل إسقاطاً مختلفاً (Random Projection) للإشارة الداخلة. الناتج: **بصمة عالية الأبعاد** (High-Dimensional Fingerprint) لكل عينة. التصنيف يتم إما بـ:
- **مسافة Hamming** إلى نماذج الفئات (Class Templates) — بسيط، مناسب لعدد فئات قليل
- **Linear Readout** (W_out @ fingerprint) — للمهام الأكبر مثل 10 فئات

أساسي: `F("0") ≠ F("1")` — مذبذبات مختلفة تستجيب بطرق مختلفة لكل فئة.

## 🏛 2. المعمارية

```
                     ┌─────────────┐
  Image (784 px) ───→│ Random Proj │──→ proj[0..N-1]
                     │ (N × 784)   │
                     └─────────────┘
                            │
          ┌─────────────────┼──────────────────┐
          │ proj[0]         │ proj[1]  ...     │ proj[N-1]
          ▼                 ▼                   ▼
     ┌─────────┐      ┌─────────┐         ┌─────────┐
     │ NCO #0  │      │ NCO #1  │   ...   │ NCO#N-1 │
     │(LFSR+ACC)│     │(LFSR+ACC)│         │(LFSR+ACC)│
     └────┬────┘      └────┬────┘         └────┬────┘
          │ fp[0]          │ fp[1]            │ fp[N-1]
          └────────────────┼──────────────────┘
                           │
                      ┌────▼────┐
                      │  Vote / │
                      │  Readout│
                      └────┬────┘
                           │ class_id
```

### 2.1 المكونات

| المكون | لكل NCO | لـ N=256 |
|--------|:-------:|:--------:|
| Accumulator 32-bit signed | 32 LCs | 8,192 |
| LFSR 32-bit (Galois) | 32 LCs | 8,192 |
| مقارن عتبة (±TH) | 4 LCs | 1,024 |
| عداد إطلاقات 16-bit | 8 LCs | 2,048 |
| **NCO sub-total** | **~76 LCs** | **~19,456 LCs** |
| Voting (Hamming/Template) | — | ~1,000 LCs |
| **الإجمالي** | | **~20K LCs** |
| | | = 20,000 LUT4s (ECP5-45K) |

### 2.2 توزيع LFSR

كل NCO يحتوي على LFSR خاص به (32-bit Galois). لكل LFSR seed مختلف:
```verilog
localparam [31:0] SEED = 32'hACE1_42BD + (i * 32'h1000_0000);
```
يضمن هذا تنوع الضوضاء بين المذبذبات — بدون تنوع، جميعها تستجيب بنفس الطريقة.

### 2.3 Random Projection (على مضيف)

المصفوفة **W_proj** (N×784) تُحسب خارج الشريحة (على مضيف مثل Raspberry Pi). يتم تحميلها عبر SPI أو UART قبل بدء التصنيف. على الشريحة: فقط NCOs + عدّادات الإطلاقات.

لتنفيذ Random Projection على FPGA نفسها (للاستقلالية الكاملة):
- جدول 784 بكسل في Block RAM
- استخدم MAC متسلسل مع معاملات مخزنة — يستهلك ~1,500 LUTs إضافية

## 📊 3. النتائج المتوقعة (من Python prototype)

| N | الفئات | الطريقة | الدقة المتوقعة |
|:-:|:------:|:-------:|:-------------:|
| 128 | 0 vs 1 | Hamming | ~95% |
| 256 | 0 vs 1 | Hamming | **~100%** |
| 512 | 0 vs 1 | Hamming | ~100% |
| 256 | 0,1,6 | Hamming | ~85-90% |
| 256 | 0,1,6 | Linear | ~90%+ |
| 128 | 10 classes | Linear | ~50-70% |
| 512 | 10 classes | Linear | ~70-85% |

→ **N=256 + Linear Readout** هو أفضل توازن بين الدقة والموارد.

## 🚫 7. قيود معروفة

1. **Random Projection خارج الشريحة** — الأوزان لا تتغير أثناء التشغيل. التعلم (بناء templates) يتم على مضيف خارجي ثم يُحمّل.
2. **N=128 مع Hamming فقط** — للـ 10 فئات، التمايز بين fingerprints غير كافٍ. يجب الانتقال إلى Linear Readout.
3. **عرض البصمة (N bits) يحد من عدد الفئات** — مع 10 فئات، تحتاج N >> 10 للحصول على فضاء فراغي كافٍ.
4. **N_WINDOW الثابت** — إذا تغير N_WINDOW، يجب إعادة بناء القوالب. هذا متوقع.

## ✅ 8. خلاصة للخبير

**HD NCO Array (Onyx V4) يثبت أن:**
- NCOs متعددة الاستجابة لإسقاطات مختلفة → بصمة فريدة لكل فئة
- Hamming Distance كافٍ لـ ≤3 فئات
- Linear Readout ضروري لـ 10 فئات
- N=256 مع ECP5-45K هو الحد الأدنى القابل للتطبيق لـ 10 فئات

**الخطوة التالية:** النموذج الأولي Python على Codespace يعطي الأرقام النهائية. إذا حقق ≥70% على 10 فئات، ننتقل إلى Verilog.
