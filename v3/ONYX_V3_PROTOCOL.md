# 🧠 Onyx V3 — Staircase Threshold Comparator Protocol

**إطار معرفي مفتوح للتصنيف متعدد الفئات بمقارن عتبات متدرج حتمي**

**5 فئات · 100% @ σ≤0.02 · 96% @ σ=0.06 · دورة ساعة واحدة · < 1 fJ**

---

## 📋 القسم 1: الملخص التنفيذي

**Onyx V3** هو الجيل الثالث من معالجات Onyx، مصمم خصيصاً للتصنيف متعدد الفئات (Multi‑class Classification). على عكس v1 و v2 اللذان يعتمدان على ذاكرة زمنية (LIF/NCO)، فإن v3 هو **مقارن عتبات متدرج حتمي (Staircase Threshold Comparator)** — لا يحتاج مذبذبات ولا ضوضاء ولا تكامل زمني.

### المبدأ

```
لكل فئة: نطاق [lo, hi) محدد مسبقاً.
الإشارة الداخلة ← تُقارن مع كل نطاق ← الفئة الأولى التي تحتوي الإشارة هي الفائزة
```

### النتائج الرئيسية

| الضوضاء σ | الدقة | الحالة |
|-----------|-------|--------|
| 0.00 | 100.0% | ✅ مثالي |
| 0.02 | 100.0% | ✅ مثالي |
| 0.04 | 99.7% | 🟡 مقبول |
| 0.06 | 95.8% | 🟡 مقبول |
| 0.10 | 81.2% | 🔴 محدود |
| 0.15 | 65.3% | 🔴 محدود |

### المقارنة مع v1 و v2

| | v1 LIF | v2 NCO | v3 Staircase |
|--|--------|--------|--------------|
| **فئات (classes)** | 2 | 2 | **5** |
| **عدد الخطوات** | 5 | 2 | **1** |
| **الطاقة/تصنيف** | 0.87 pJ | ~0.03 pJ | **< 1 fJ** |
| **دقة @ σ=0.02** | 100% | 100% | **100%** |
| **دقة @ σ=0.06** | 100% | 100% | **~96%** |
| **ذاكرة زمنية** | نعم | نعم | **لا** |
| **نوع المذبذب** | LIF | NCO | **لا يوجد** |

- ✅ 5 مقارنات فقط — يمكن تنفيذها في < 100 LUT على أي FPGA

## 📋 القسم 3: العمارة

### 3.1 Staircase Comparator

```
signal_in ──┬──> Comparator 0 (th_lo=0.00, th_hi=0.20) ──> class 0?
            ├──> Comparator 1 (th_lo=0.20, th_hi=0.42) ──> class 1?
            ├──> Comparator 2 (th_lo=0.42, th_hi=0.675) ──> class 2?
            ├──> Comparator 3 (th_lo=0.675, th_hi=0.95) ──> class 3?
            └──> Comparator 4 (th_lo=0.95, th_hi=1.65) ──> class 4?

Output: أول مقارن تكون شرطه صحيحاً → class_id
```

### 3.2 خصائص

| الخاصية | القيمة |
|---------|--------|
| عدد الفئات | 5 (قابلة للزيادة) |
| زمن التصنيف | دورة ساعة واحدة |
| الطاقة | < 1 fJ (مقارنات فقط) |
| المذبذبات | صفر |
| الضوضاء | غير مطلوبة |
| العتبات | ثابتة، محسوبة من توزيع الفئات |

### 3.3 آلية العتبات

العتبات تحسب تلقائياً من مراكز الفئات:
```
فئة i: [lo, hi) حيث
  lo = (center[i-1] + center[i]) / 2
  hi = (center[i] + center[i+1]) / 2
الاستثناءات:
  الفئة الأولى: lo = 0.0
  الفئة الأخيرة: hi = center × 1.5
```

## 📋 القسم 4: القيود المعروفة

### 4.1 حدود v3

1. **الضوضاء:** الدقة تتناقص مع الضوضاء. عند σ=0.10، الدقة 81%. غير مناسب للتطبيقات عالية الضوضاء.

2. **حتمي:** لا توجد ذاكرة زمنية — عينة واحدة تحدد القرار. لو العينة مشوشة بشدة، القرار خطأ.

3. **عتبات ثابتة:** لا يمكن التكيف مع تغير توزيع الإشارات. نظام التكيف التجريبي لم يحسن النتائج.

### 4.2 متى تستخدم v3

| الحالة | الاستخدام |
|--------|-----------|
| ضوضاء σ ≤ 0.02 | ✅ مثالي — 100% دقة |
| ضوضاء σ ≤ 0.06 | 🟡 مقبول — 95-99% |
| ضوضاء σ > 0.06 | ❌ استخدم v1/v2 (ثنائي) أو معماريّة مختلفة |

### 4.3 لماذا v3 أقل من v1/v2 في الضوضاء؟

```
v1/v2 (LIF/NCO): x(t) → ∫ عبر N_STEPS → فقط ← ذاكرة زمنية تفلتر الضوضاء
v3 (Staircase):  x(t) → مقارنة لحظية    ← عينة واحدة، ضوضاء غير مفلترة
```
